# Brainlabs Brand Acceptance Checklist

Use this checklist after importing official Brainlabs brand assets.

## Visual Tokens
- Replace placeholder colors in `web/static/brand/tokens.css` with official values.
- Replace placeholder fonts in `web/static/brand/tokens.css` with licensed Brainlabs typography.
- Verify spacing, radii, and elevation values match design system tokens.

## Components
- Buttons (`primary`, `secondary`, `ghost`, `danger`) match approved sizing and states.
- Input fields (default/focus/error) match approved styles.
- Status badges align with approved semantic palette.
- Tables match approved typography, row density, and divider styles.

## Layout & Responsiveness
- Desktop and mobile spacing align with approved breakpoints.
- Header, navigation, and hero pattern matches approved brand layout.
- Dashboard KPI cards and detail panels preserve visual hierarchy on all viewport widths.

## Assets
- Replace placeholder text logo with official Brainlabs SVG in top navigation.
- Add favicon and social sharing assets.
- Verify any iconography is from approved icon set.

## QA Sign-Off
- Marketing/design sign-off completed.
- Internal stakeholder sign-off completed.
- Screenshots stored for baseline comparison.
